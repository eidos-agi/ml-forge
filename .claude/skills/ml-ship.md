# ml-ship — Package Model as Static Artifact

Package a trained model as a minimal, zero-dep artifact that integrates into the target project.

## Trigger

User says `/ml-ship` or has a trained model ready to integrate.

### Arguments

- `<project>` — the target project
- `<weights_file>` — path to the model weights JSON from /ml-train

## Instructions

### Step 1: Choose Artifact Format

Based on model type and target project constraints:

| Model | Artifact | Runtime code |
|-------|----------|-------------|
| Logistic regression | JSON weights file or inline constants | `sigmoid(dot(weights, features) + intercept)` |
| Decision tree | JSON rules or generated if/else | Tree traversal function |
| Random forest | JSON rule set | Majority vote over trees |
| Simple scorer | Python dict of learned weights | Weighted sum |

**Prefer inline constants** (hardcoded in source) over separate JSON files when the model is small (<50 weights). This avoids file loading and makes the model visible in code review.

### Step 2: Generate Runtime Module

Generate from `model.py.tmpl` — a single Python file with:
- The model weights as constants at module level
- A `predict(features: dict) -> float` function
- A `predict_label(features: dict) -> str` function (thresholded)
- No imports beyond stdlib (math, json, typing)

Example for logistic regression:

```python
"""Auto-generated by ml-forge. Do not edit weights manually — retrain instead."""
import math

# Learned weights — trained on N examples, accuracy: X%
# See train_model.py for reproduction
WEIGHTS = [0.42, -0.18, 0.91, ...]  # feature_name_1, feature_name_2, ...
INTERCEPT = -1.23
THRESHOLD = 0.5
FEATURE_NAMES = ["feature_name_1", "feature_name_2", ...]

def predict(features: dict) -> float:
    """Return probability score (0-1)."""
    vec = [features.get(name, 0.0) for name in FEATURE_NAMES]
    logit = sum(w * x for w, x in zip(WEIGHTS, vec)) + INTERCEPT
    return 1.0 / (1.0 + math.exp(-logit))

def predict_label(features: dict) -> bool:
    """Return True/False prediction."""
    return predict(features) >= THRESHOLD
```

### Step 3: Integration

1. Copy the runtime module into the target project
2. Wire it into the existing code path (replace hardcoded logic or add as a new signal)
3. Keep the hardcoded fallback — if the model has low confidence, fall back to rules

### Step 4: Document

Add to the target project's CLAUDE.md or inline comments:
- What the model predicts
- How it was trained (pointer to training script)
- When to retrain (new labeled data, accuracy drift)
- The baseline it beat

## Output

- Runtime inference module (stdlib only, no deps)
- Integration guidance for the target project
- Retraining instructions

## Rules

- **Zero runtime deps** — the shipped module must use only stdlib (GUARD-004)
- **Weights are constants** — embedded in source or loaded from a committed JSON file
- **Always include a fallback** — the model augments existing logic, doesn't replace it entirely
- **Comment the provenance** — training date, accuracy, data size, so future devs know what they're looking at
- **Never ship the training script** — it stays in dev, uses dev-only deps
