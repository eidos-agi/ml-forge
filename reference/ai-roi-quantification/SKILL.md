# AI ROI Quantification
## When to Use This Skill
Use when you need to quantify AI's value: justifying investment, reporting to stakeholders, pitching clients, or deciding which AI projects to fund.

## Instructions
1. **Baseline Measurement (Before AI)**
   - Time per task (hours)
   - Cost per task (labor × hourly rate)
   - Error rate (% errors, rework rate)
   - Throughput (tasks per day/week)
   - Revenue/outcome (if applicable)
2. **Post-AI Measurement**
   - Same metrics after AI implementation
   - Track for minimum 30 days for reliable comparison
   - Control for other changes (seasonality, staffing, process changes)
3. **ROI Calculation**
   - Time savings: (hours_before - hours_after) × tasks_per_year = annual hours saved
   - Cost savings: annual_hours_saved × hourly_rate = annual cost savings
   - Error reduction: (error_rate_before - error_rate_after) × cost_per_error × volume
   - Revenue impact: new_revenue_attributable_to_AI (harder to isolate but most powerful)
   - Total ROI = (total_benefits - total_costs) / total_costs × 100%
4. **Attribution** — Be honest about what AI caused vs. what happened concurrently. Use before/after, control groups, or A/B tests when possible.
5. **Presentation Frameworks**
   - For executives: "AI saved $X and Y hours, enabling Z new capability"
   - For clients: "Your current process costs $X/year; with AI it costs $Y/year; payback in Z months"
   - For investors: "AI creates N basis points of alpha, reduces operational risk by X%"
6. **Compounding Effects** — First-order: time/cost savings. Second-order: freed capacity enables new projects. Third-order: competitive moat from AI capabilities others don't have. Document all three levels.
7. **Common Pitfalls** — Don't count savings from work that wouldn't have been done anyway. Don't double-count benefits. Include implementation costs (your time, API fees, infrastructure). Be conservative — credibility > impressive numbers.
